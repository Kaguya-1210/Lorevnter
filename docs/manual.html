安装

打开酒馆助手

在 SillyTavern 中打开酒馆助手面板。

导入脚本

点击「导入脚本」，选择 Lorevnter.js 文件导入即可。

确认安装

导入成功后，顶部工具栏出现 Lorevnter 按钮即安装完成。无需重启。

🚀

快速上手

打开插件窗口

点击顶部工具栏的 Lorevnter 按钮，或在聊天框输入 /lore。

确认世界书识别

窗口底部的状态栏会显示当前角色卡绑定的世界书。确保你的角色卡正确配置了世界书。

设置约束（推荐）

在「约束管理」页签中为条目创建约束。约束告诉 AI 你想要怎样更新这个条目——比如"记录能力值变化"或"跟踪人际关系发展"。

配置触发方式

在「AI 配置」页签设置自动触发间隔（每 N 轮 AI 回复后自动分析），或选择手动模式。

开始游玩

正常聊天即可。AI 会在合适时机分析剧情变化并弹出修改建议。审核通过后自动写入世界书。

⚙️

工作原理

管线流程

Lorevnter 的核心是一条自动化管线，每次触发时按以下步骤执行：

触发分析
→
自动备份
→
收集条目
→
提取正文
→
AI 分析
→
审核确认
→
写入世界书

触发方式

方式说明
自动每 N 轮 AI 回复后自动触发（在 AI 配置中设置间隔）
手动在世界书页签点击「🔍 AI 分析」按钮
斜杠命令在聊天框输入 /lore 打开插件窗口后手动触发
💡 建议从间隔 3-5 轮开始，根据剧情节奏调整。高频分析消耗更多 API 额度但信息更及时。
🔒

约束系统

什么是约束？

约束是你给 AI 的条目级指令。每个约束可以绑定到一个或多个世界书条目，告诉 AI 在分析该条目时应该遵循的规则。

约束类型

类型标签作用
提示词约束PROMPT告诉 AI 如何处理该条目（记录什么、格式要求、更新策略等）
跳过约束SKIP标记条目不参与 AI 分析（固定设定、模板类条目）

约束作用域

作用域说明
全局所有角色卡通用，切换角色不影响
局部仅对创建时的角色卡生效，切换角色时自动隔离

约束编写示例

角色能力值追踪 
记录 {{char}} 的能力值变化和状态数据。
当正文中出现能力值、等级、HP、MP 等数值变化时，更新对应字段。
可以自由扩展条目，添加新发现的能力和技能。
格式要求：使用「属性名: 数值」的格式。
人际关系记录 
追踪 {{char}} 与其他角色之间的关系发展。
记录关系变化的原因和具体事件。
包括好感度、信任度等定性描述。
新出现的重要 NPC 应新增关系条目。
世界设定记录 
记录该地点/组织的最新状态。
关注：成员变动、势力范围、重大事件、政策变化。
保持条目简洁，只记录已确认的事实。

约束优先级

当约束与系统默认策略冲突时，约束始终优先。优先级从高到低：

  1. 输出格式规则（必须输出有效 JSON）
  2. 数据安全规则（不丢失未变化的内容）
  3. 局部约束（角色卡级）
  4. 全局约束（跨角色卡）
  5. 默认更新策略
ℹ️ 约束说"自由扩展"，AI 就会执行扩展——即使默认策略倾向保守。约束的力量高于一切默认行为。
🏷️

宏系统

内置宏

在约束文本和提示词模板中，可以使用以下宏：

宏说明使用场景
{{char}}当前角色名约束、提示词
{{user}}用户名约束、提示词
{{lore_entries}}待分析条目列表提示词模板
{{lore_context}}聊天正文（提取后）提示词模板
{{lore_constraints}}所有约束规则提示词模板
{{lore_user_persona}}用户人设提示词模板
{{lore_constraint_policy}}约束优先级规则提示词模板

条目级宏 CUSTOM

你可以为约束设置 macro 字段，创建条目级别的短标签别名。在提示词模板中通过 {{别名}} 引用条目内容。

例如：为角色「沙赫尔」的约束设置 macro 为 shr，即可在任何提示词中用 {{shr}} 引用该条目的完整内容。

提示词编辑

提示词结构

发送给 AI 的提示词由以下部分组成（按顺序）：

#角色内容
1system系统规则 + 约束 + 决策规则
2user用户人设数据
3user世界书条目列表
4user聊天正文
5userCoT 思考指令
6assistantCoT 锚定（引导开始思考）

自定义提示词

在「预设管理」页签中可以管理提示词预设。每个预设包含多个 Prompt Item，你可以：

  • 调整每个 Item 的角色（system / user / assistant）
  • 编辑内容模板，使用宏动态注入数据
  • 拖拽排序、启用/禁用单个 Item
  • 创建多个预设并快速切换
⚠️ 修改提示词需要了解 AI 的 prompt 工程原理。建议在调试面板的「管线预览」中查看最终发送的内容，确认修改效果。
✅

审核机制

审核流程

AI 分析完成后会弹出审核面板，每条修改建议包含：

  • 条目名 — 要修改哪个条目
  • 操作类型 — 修改 / 追加 / 新增
  • AI 理由 — 为什么要这样改
  • Diff 对比 — 展开查看逐行差异（红色=删除，绿色=新增）

审核操作

操作说明
✅ 通过确认该条修改
❌ 拒绝拒绝该条修改
全部通过一键通过所有待审核项
确认执行将所有已通过的修改写入世界书
取消放弃所有修改
💡 调试模式下会强制开启审核，即使全局审核开关关闭。建议初期开启审核以建立信任。
👤

用户人设

人设处理功能

在「AI 配置」页签中开启「用户人设处理」后，AI 分析时会：

  • 读取酒馆的用户 Persona 信息
  • 将其作为 <user_persona> 标签注入提示词
  • AI 可以提出人设更新建议（以 __persona__ 条目名输出）
  • 审核通过后自动写回酒馆的 Persona 设置

人设约束绑定

在「约束管理」页签中，选中一个约束后可以点击「绑定人设」按钮，将该约束作为人设处理的专属规则。

ℹ️ 系统默认规则会限制人设更新的范围——只记录用户自身的稳定设定和长期偏好，不会写入他人状态或世界设定。约束可以覆盖此行为。

条目过滤

过滤模式

模式说明
全部分析所有世界书条目（默认）
仅处理指定只分析你勾选的条目
排除指定分析除你勾选外的全部条目

过滤操作

  • 🔍 搜索 — 快速定位条目
  • ☑️ 全选/清空 — 批量操作
  • 🔄 反选 — 反转当前选择
  • 📌 搜索结果全选 — 只选中搜索结果中的条目
💡 切换角色卡时，过滤设置会自动重置为"全部"模式，避免跨角色误用。
💾

缓存管理

SCAN_DONE 缓存

Lorevnter 监听酒馆的 SCAN_DONE 事件，缓存当前激活的世界书条目。管线分析时优先处理缓存中的命中条目，提高效率。

清理模式

模式说明
分析后自动每次管线完成后自动清空缓存（推荐）
手动缓存持续累积，需要手动清理
ℹ️ 超过 3 天未更新的缓存会在切换聊天时弹出提醒，可一键清理。
📋

备份与回档

自动备份

每次 AI 分析前，Lorevnter 会自动为所有参与的世界书创建快照。如果 AI 的修改不理想，可以一键回档。

  • 备份间隔 — 每 N 次管线执行备份一次（默认每次）
  • 保留数量 — 最多保留 N 个备份（默认 5 个）
  • 回档操作 — 在世界书页签的备份列表中选择并还原
🐛

调试面板

调试工具一览

工具功能
系统自检检查 API 连通性、世界书状态、设置完整性等
管线预览查看发送给 AI 的完整提示词和 Token 统计
AI 历史查看历次 AI 分析的请求和响应
操作日志实时日志，支持级别过滤（info / warn / error）
测试模式模拟写入假数据测试审核/写入管线
缓存面板查看当前缓存的命中条目
一键导出导出完整调试信息用于排查问题

测试模式

在调试面板中可以使用测试模式：

  1. 选择目标世界书
  2. 点击「执行测试写入」— 插件会构造模拟数据并打开审核弹窗
  3. 审核确认后写入世界书
  4. 点击「一键回档」恢复到测试前状态
💡 测试模式不消耗 API 额度，适合验证审核/写入管线是否正常工作。

常见问题

AI 分析后什么都没更新？

可能的原因：

  • 没有设置约束 — AI 在无约束时倾向保守，建议为重要条目设置提示词约束
  • 正文太短 — AI 需要足够的聊天上下文才能发现变化
  • 条目被 skip — 检查是否有跳过约束绑定
  • 缓存为空 — 首次运行无缓存会分析全部条目，确认正文中提到了相关角色/地点
AI 只更新人设，不更新世界书条目？

这通常是因为缺少 CoT 思考链引导。Lorevnter 内置了 CoT 指令，确保 AI 逐条分析每个条目而不只是关注最显眼的变化。如果仍有问题：

  • 确认约束中明确了需要记录的内容类型
  • 在调试面板的「管线预览」中检查提示词是否正确
  • 尝试调低 Temperature 提高 AI 遵循指令的准确性
缓存越来越大怎么办？

Lorevnter 的缓存策略是覆盖模式——每次 SCAN_DONE 事件用最新的激活列表直接覆盖，不会无限累积。如果使用「分析后自动」清理模式（默认），每次管线完成后缓存会自动清空。如果选择了「手动」模式，可以在 AI 配置页手动清理。

怎么知道 AI 看到了什么？

在调试面板中点击「管线预览」，可以看到发送给 AI 的完整提示词内容，包括：

  • 系统规则和约束
  • 待分析条目列表（含当前内容）
  • 提取后的聊天正文
  • Token 统计（确保不超过模型上下文限制）
支持哪些 AI 模型？

Lorevnter 使用 OpenAI 兼容的 API 格式。支持：

  • OpenAI GPT-4o / GPT-4
  • Claude (通过兼容 API)
  • 任何 OpenAI 兼容端点（如 OpenRouter、本地部署等）

复用酒馆连接时，使用酒馆当前配置的模型。

约束和提示词的区别？

约束是条目级的规则——针对特定条目的处理指令。提示词是全局级的模板——控制 AI 的整体分析行为和输出格式。两者协同工作：提示词定义了分析框架，约束在框架内为每个条目定制策略。
