# 📘 Lorevnter 使用指南

> AI 驱动的世界书自动管理插件
> 让世界观数据随剧情自动进化

---

## 📦 安装

酒馆助手 → 导入脚本 → 选择 `Lorevnter.js` → 完成  
顶部工具栏出现 **Lorevnter** 按钮即安装成功

---

## 🚀 快速上手

### 第一步：打开插件
- 点击顶部 **Lorevnter** 按钮
- 或聊天框输入 `/lore`

### 第二步：确认世界书
窗口底部状态栏会显示当前角色卡绑定的世界书  
确保你的角色卡已配置世界书

### 第三步：设置约束（重要！）
进入「约束管理」页签，为你的条目创建约束  
约束决定了 AI 怎么分析和更新你的条目

### 第四步：配置触发
「AI 配置」页签中设置：
- **自动模式**：每 N 轮 AI 回复后自动分析
- **手动模式**：需要时手动点击分析

### 第五步：游玩
正常聊天即可。AI 会自动分析并弹出修改建议  
审核通过后写入世界书 ✅

## 🔒 约束系统

约束是这个插件最核心的功能——**告诉 AI 你想怎么更新条目**

### 约束类型

| 类型 | 作用 |
|:---:|:---|
| **提示词** | 告诉 AI 如何处理条目 |
| **跳过** | 标记条目不参与分析 |

### 约束作用域

| 作用域 | 说明 |
|:---:|:---|
| **全局** | 所有角色卡通用 |
| **局部** | 仅当前角色卡生效 |

### 怎么写约束？

**约束拥有最高权力**。你在约束里写什么，AI 就执行什么

#### 📝 示例 1：能力值追踪
```
记录 {{char}} 的能力值变化和状态数据。
出现数值变化时更新对应字段。
可以自由扩展，添加新发现的能力和技能。
格式：属性名: 数值
```

#### 📝 示例 2：人际关系
```
追踪 {{char}} 与其他角色的关系发展。
记录关系变化原因和具体事件。
新出现的重要 NPC 应新增关系条目。
```

#### 📝 示例 3：世界设定
```
记录该地点/组织的最新状态。
关注：成员变动、势力范围、重大事件。
只记录已确认的事实。
```

> 💡 **关键词**：约束里写了"自由扩展"，AI 就会扩展条目内容；写了"记录数值"，AI 就会更新数值。约束说什么 AI 就做什么，不用担心被默认策略拦住。

## 🏷️ 宏系统

在约束和提示词中可以用宏自动替换变量

### 常用宏

| 宏 | 含义 |
|:---|:---|
| `{{char}}` | 当前角色名 |
| `{{user}}` | 你的用户名 |
| `{{lore_entries}}` | 待分析条目 |
| `{{lore_context}}` | 聊天正文 |
| `{{lore_constraints}}` | 约束规则 |
| `{{lore_user_persona}}` | 用户人设 |

### 条目级宏

可以给约束设置 macro 字段创建短标签  
例如：macro = `shr` → 在提示词中用 `{{shr}}` 引用该条目内容

## ✅ 审核机制

AI 分析完成后弹出审核面板

每条建议包含：
- 📌 条目名 + 操作类型（修改/追加/新增）
- 💬 AI 的修改理由
- 📊 展开查看 Diff 对比（红删绿增）

操作：
- ✅ 通过 / ❌ 拒绝 — 逐条审核
- 全部通过 — 一键全过
- 确认执行 — 把通过的写入世界书
- 取消 — 放弃所有修改

> 💡 调试模式下会强制开启审核，建议初期先开着建立信任

---

## 👤 用户人设

在「AI 配置」开启「用户人设处理」后：

1. AI 读取你的酒馆 Persona 信息
2. 作为 `<user_persona>` 注入提示词
3. AI 可以建议人设更新
4. 审核通过后写回酒馆 Persona

在「约束管理」中可以给人设绑定专属约束

---

## 🔍 条目过滤

控制哪些条目参与分析

| 模式 | 说明 |
|:---:|:---|
| **全部** | 分析所有条目（默认） |
| **仅指定** | 只分析你勾选的 |
| **排除指定** | 排除勾选的，分析其余 |

支持：搜索、全选、清空、反选

> 切换角色卡时过滤自动重置

---

## 💾 缓存管理

插件会缓存酒馆激活的世界书条目，分析时优先处理缓存命中的条目

**清理模式：**
- 分析后自动清空（推荐，默认）
- 手动清理

> 超过 3 天未更新的缓存会自动提醒

---

## 📋 备份与回档

每次 AI 分析前自动为世界书创建快照

在「世界书」页签可以查看备份列表并一键回档

设置项：
- 备份间隔（每 N 次执行备份一次）
- 最大保留数（默认 5 个）

---

## ✏️ 提示词编辑

> ⚠️ 高级功能，新手可以跳过

发送给 AI 的消息顺序：

```
1. [system] 系统规则 + 约束 + 决策规则
2. [user]   用户人设数据
3. [user]   世界书条目列表
4. [user]   聊天正文
5. [user]   CoT 思考指令
6. [assistant] CoT 锚定
```

在「预设管理」中可以自定义提示词模板  
修改前建议在调试面板的「管线预览」中查看最终内容

## ❓ 常见问题

### AI 什么都没更新？
- 给条目设了约束没？没约束 AI 会偏保守
- 正文太短？AI 需要足够上下文
- 条目被 skip 了？检查有没有跳过约束

### AI 只改人设不改条目？
- 确认约束里写清楚了要记录什么
- 在管线预览中检查提示词
- 试试调低 Temperature

### 缓存一直有东西？
- 默认模式是"分析后自动清空"
- 分析完有 3 秒冷却，期间不会重新缓存
- 手动模式需要自己清

### 支持哪些模型？
OpenAI 兼容格式的都行：
- GPT-4o / GPT-4
- Claude（通过兼容 API）
- OpenRouter、本地部署等
